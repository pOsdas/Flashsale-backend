# Demo-сценарий

Этот документ описывает рекомендуемый сценарий демонстрации проекта.

Цель демо — показать полный жизненный цикл мониторинга товара: от добавления товара до отправки уведомления пользователю

## Подготовка

Перед демонстрацией убедитесь, что:

- проект запущен;
- parser-health отвечает;
- Grafana доступна;
- RabbitMQ доступен;
- Telegram-канал подключен (если демонстрируются уведомления).

Проверка parser-health:

```bash
curl -X GET "http://127.0.0.1:8090/api/v1/parser/health"
```

## Шаг 1. Архитектура

```text
 Monitoring Target
        ↓
 Monitoring Scanner
        ↓
  Product Cache
        ↓
    go_fetcher
        ↓
 Product Snapshot
        ↓
      Alert
        ↓
      Outbox
        ↓
     RabbitMQ
        ↓
Notification Consumer
        ↓
     Telegram
```

- scanner проверяет товары;
- fetcher получает данные;
- backend сравнивает изменения;
- при изменениях создается alert;
- alert превращается в уведомление.

## Шаг 2. Parser-health

```bash
curl -X GET "http://127.0.0.1:8090/api/v1/parser/health"
```

- что система умеет диагностировать парсеры;
- что отдельно проверяются WB и Ozon;
- что видно наличие cookies и тип ошибок.

## Шаг 3. Создание Monitoring Target

Создать новый объект мониторинга через API или административный интерфейс.

Показать:

- marketplace;
- URL товара;
- интервал проверки.

После создания target должен попасть в очередь проверок scanner-а.

## Шаг 4. Работа Scanner

Показать логи scanner-а:

```bash
docker compose logs -f monitoring_scanner
```

- scanner выбирает товары, которые пора проверить;
- использует общий кэш;
- обращается к fetcher только при необходимости;
- создает новый snapshot состояния товара.

## Шаг 5. Product Snapshot

Показать сохраненные snapshot-ы товара.

Обратить внимание на:

- цену;
- наличие;
- рейтинг;
- количество отзывов;
- время проверки.

## Шаг 6. Alert

После изменения данных товара показать созданный alert.

Например:

```text
Цена изменилась
Товар появился в наличии
Изменился рейтинг
Появились новые отзывы
```

## Шаг 7. RabbitMQ

Открыть RabbitMQ:

```text
http://127.0.0.1:15672
```

Сначала создается событие:

```text
Alert → Outbox → RabbitMQ
```

## Шаг 8. Notification Consumer

- consumer получает событие;
- формирует уведомление;
- отправляет его пользователю.

## Шаг 9. Показать Telegram уведомление

Если Telegram подключен:

Приходят уведомления в Telegram.

Например:

```text
Цена товара изменилась

Было: 4990 ₽
Стало: 4490 ₽

Снижение: 500 ₽
```

## Шаг 10. Observability

Grafana:

```text
http://127.0.0.1:3000
```

- метрики backend;
- метрики outbox worker;
- метрики notification consumer;
- логи через Loki.
